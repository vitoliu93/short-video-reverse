# Goal: 视频文字「字体 + 样式」识别 — 算法研究与工程实验

**Created**: 2026-05-29  **Tier**: full

## Intent
短视频反解管线（`short-video-reverse`）中，§3「字体/样式识别」是研究文档点名的**最薄弱环节**。
本任务把它从开放难题收敛为**闭集匹配**：因为我们拥有全部字体 TTF 文件（`tos://kox-statics/fonts_effect/`，~1300+ 字体），
问题变成「给定渲染后视频里的一块文字区域，判断它用的是**我们字体库里的哪一款**字体，以及它的视觉样式」。
产出服务于 KOX eval Gold Case 与 `add_text` 的 font / color / transform 参数还原。

## Done means
- **算法研究结论**：用证据回答「闭集字形渲染-比对能否在 ~1300 个 CJK 字体里把正确字体放进 top-K」，并给出推荐生产方案（含精度、成本、降级路径）。
- **可跑的 POC**：一个实验性 Python 模块，输入「视频里的文字区域」→ 输出结构化 `texts[]` JSON：
  - **字体**：top-1 库内字体名 + topk + 置信分 + 粗类（黑/宋/手写/圆/书法…）
  - **颜色与修饰**：填充色 HEX、是否渐变、描边色+宽度、阴影
  - **字重与字号**：bold/regular/thin、相对字号（文字高 / 画面高）
  - **位置与动画**：bbox 对齐位（底部居中/顶部/居中大字…）、入场动画（none/fade/pop/scroll/typewriter）
- **验证通过**：
  - 合成集（用已知库内字体渲染 → 看匹配器能否找回）给出可量化的 top-1/top-5 精度（字体精度只能靠合成集量化，真实抖音无可知 ground-truth）。
  - 真实抖音样本（3–5 条，自行采集）上端到端跑出 JSON，样式属性对眼标可核对，字体匹配「视觉合理性」可接受。

## Explicitly out of scope
- **不做生产管线集成**：停在「POC 在测试集上 work」，不接入 `short-video-reverse` 端到端编排（对应 BGM 的 M3，本任务不含）。
- **不自研 OCR/检测**：文字检测+识别（bbox + 文本）借用现成 OCR（RapidOCR/PaddleOCR）作为 POC 前端，不是研究重点；EVE/字幕时间轴由其他模块负责。
- **不做开放集字体命名**（识别世界上任意字体）——只匹配到我们字体库内。
- **不做精确到像素的动画曲线还原**——动画只做类型分类，必要时降级为「静态/动态」二分。

## Scope changes (append-only)
- 2026-05-29 立项。交付=研究+POC；样式四维全覆盖；eval 用自采真实抖音样本（字体精度辅以合成集量化）。批准人：vito（AskUserQuestion 确认）。
