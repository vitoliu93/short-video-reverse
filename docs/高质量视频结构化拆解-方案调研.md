# 高质量视频结构化拆解 — 方案调研

## 问题定义

**场景**：用户提供一个抖音/短视频 URL（仅像素，无工程文件），需要反向拆解出视频的编辑结构，作为 KOX eval set 的"黄金标准"数据来源。

**拆解目标**：从渲染后的视频像素中提取：

| 编辑元素 | 对应 KOX draft 参数 |
|-|-|
| 镜头切分（时间线） | add_video 的 start/end 分段 |
| 字幕/文字内容、位置、样式 | add_text 的 text / transform / font / color |
| BGM / 音效 | add_audio 的 audio_url / volume / start/end |
| 转场类型 | add_video 的 transition |
| 特效/滤镜 | add_effect / add_filter |
| 整体叙事结构 | 脚本分段 + 素材编排逻辑 |

---

## 核心发现：ARC-Hunyuan-Video-7B（腾讯，2025.07）

**最接近需求的开源模型。** 专门为抖音/微信视频号短视频理解设计，7B 参数，已开源。

**关键能力**：

- 同时处理视觉 + 音频 + 文本三模态
- 帧上叠加时间戳（HH:MM:SS）实现精确定位
- 多粒度带时间戳的描述、视频摘要、开放 QA、时序定位、视频推理
- 引入 ShortVid-Bench 评估 6 维度：时序推理、情感意图、创作者意图、叙事理解、幽默解构、创意创新
- 74.3% 准确率，超过 Qwen2.5-VL-7B
- 10 秒推理 / 1 分钟视频（H20 GPU，vLLM 加速）
- 训练管线：预训练 → 指令微调 → RL 后训练（GRPO）→ 最终微调

**与 KOX 的关系**：作为"拆解高质量视频"的主力模型，提取叙事结构、内容类型、关键时间点、情感曲线。

- 论文：https://arxiv.org/abs/2507.20939
- 模型：https://huggingface.co/papers/2507.20939

## 子任务技术栈

### 1. 镜头切分 — TransNetV2 ✅ 生产级成熟

扩张 1D 卷积处理降采样 RGB 帧，150+ FPS，输出：

- 切点时间戳列表
- 转场类型三分类（无转场 / 硬切 / 渐变中心帧）
- 后处理：最小镜头长度约束 + 滑动窗口平滑

工程依赖极轻，直接集成。

### 2. 字幕/文字提取 — EVE ✅ 有方案

复旦 + 字节跳动，arXiv:2503.04058，2025.03。

- 端到端 VLM：输入视频帧 → 输出字幕文本 + 时间戳
- 双分支架构：空间语义聚合（SSCA）+ 时序字幕 token 查询（TSTQ）
- 训练数据 ViSa：2.5M+ 视频，中英双语带时间戳标注
- 提供 7B 和 0.5B 两版本
- **局限**：目前只输出文本和时间戳，不输出字幕坐标（bounding box）

论文：https://arxiv.org/abs/2503.04058

对于字幕位置和样式，需要补充方案：

- PaddleOCR 检测每帧文字 bounding box → 计算坐标和尺寸
- 字幕区域截图 → VLM（Doubao）描述字体类型、颜色、粗细
- 规则推导动画类型（淡入、弹出、滚动等）

### 3. 字体/样式识别 — ⚠️ 最薄弱环节

目前没有专门做"视频中文字字体识别"的成熟方案。可用的替代路径：

| 降级方法 | 说明 |
|-|-|
| VLM 字体描述 | 字幕截图 → VLM 判断 "黑体/宋体/手写体/圆体" 等大类 |
| 规则取色 | OpenCV 取字幕区域主色 → RGB/HEX |
| 位置分类 | PaddleOCR bounding box → 底部居中 / 左上角 / 居中大字 等 |
| 字号估算 | bounding box 高度 / 画布高度 → 估算相对字号 |

如果需要精确到具体字体名称的匹配，需要自建 KOX 常用字体数据集做 fine-tune。

### 4. BGM 识别 — ✅ 成熟

两条路径可按需选择：

| 路径 | 工具 | 适用场景 |
|-|-|-|
| 经典音频指纹 | SeekTune（Go 实现 Shazam 算法） | 已知曲库匹配 |
| 神经指纹 | neural-music-fp（ISMIR 2025） | 噪声/变速场景鲁棒性更强 |
| BGM 风格标签 | CLAP Score 匹配 | 不需要精确曲名，只要风格分类 |

### 5. 转场/特效分类 — ⚠️ 部分覆盖

- TransNetV2 已覆盖硬切和渐变的基本分类
- 剪映风格复杂特效（抖动、模糊、闪光、故障等）需 VLM 补充：

  - TransNetV2 检测切点位置
  - 切点前后各取 0.5s 帧
  - VLM 判断转场类型名称
- 滤镜类型同理：关键帧截图 → VLM 分类描述

### 6. 整体编排 — Agentic Video Intelligence (AVI) 范式

2025 年主流做法：LLM Agent 做任务分解 → 编排工具模块 → 合成结果。不做端到端黑箱。

代表系统：

- VideoDeepResearch：Plan → Invoke → Synthesize 循环，32 帧超越端到端 MLLM
- VideoMiner（ICCV 2025）：树形 RL 从长视频逐层提取关键帧
- CAViAR：LLM Agent + Critic 增强推理

## KOX 拆解管线设计

```
抖音视频 URL
    │
    ├── ffmpeg ───────────── 抽帧 (1fps) + 提取音频
    │
    ├── TransNetV2 ───────── 镜头切分时间线 + 基础转场分类
    ├── EVE / ARC-Hunyuan ── 字幕文本 + 时间戳
    ├── PaddleOCR ────────── 字幕 bounding box → 位置 + 字号
    ├── VLM (Doubao) ─────── 字体描述 + 颜色 + 样式特征 (截图送 VLM)
    ├── 音频指纹 ─────────── BGM 曲目/风格标签
    ├── TransNetV2 + VLM ─── 转场类型 + 位置
    └── ARC-Hunyuan-Video ── 整体叙事结构 + 内容标签 + 情感曲线
                              │
                              ▼
                    结构化编辑描述 JSON
                    {
                      "shots": [{start, end, type, content}],
                      "subtitles": [{text, start, end, font_class, color, position}],
                      "bgm": {name, style, start, end},
                      "transitions": [{at_time, type}],
                      "narrative": {structure, hook_type, pacing_profile}
                    }
                              │
                              ▼
                    映射规则 → KOX icccut_draft.json 模板参数
```

## 与 KOX eval 体系的关系

拆解管线产出的是 **Gold Case 的结构化表示**，直接服务于之前两份文档定义的目标：

1. **构建 eval set**：多个人工精选的高质量视频 → 拆解为结构化 Gold Case → 提取统计基准线
2. **注入评估体系**：Gold Case 的统计分布 → Stage 2（CLIP Score 阈值）、Stage 3（节奏指标、文字时机规则）
3. **模板学习**：高频编辑模式 → 提炼为新的 pipeline 模板或优化现有模板参数

## 结论

除"视频文字字体精确识别"外，其余子任务都有可用方案。瓶颈不在模型能力，而在：

1. 多模型的工程整合和调度
2. 拆解结果到 KOX draft JSON 参数的映射规则
3. 字体识别需要自建数据集或接受 VLM 大类判断的降级方案

ARC-Hunyuan-Video-7B 是最值得深入阅读的——唯一专门针对抖音风格短视频的开源多模态理解模型。
